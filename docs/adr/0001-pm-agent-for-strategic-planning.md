# PM agent for Strategic Planning

This project is built by a solo developer in a school setting whose supervisor is
non-technical and does not act as a product manager. The obvious choice for a solo
developer would be to drop Strategic Planning (the project-level brain dump → PRD →
tickets stage) as PM overhead. We deliberately keep it, and have a **PM agent** play the
product-manager role: it turns the developer's discussions with that non-technical
stakeholder into a reviewable PRD and tickets.

Without the PM agent, requirements would stay trapped in non-technical conversation and
never reach a PRD + ticket form that Phase 2 (the PIV Loop) can consume. The PM agent fills
the missing human-PM role; this is why Strategic Planning survives here even though the
developer works alone.
